  • HTTPMethod

    An enumeration representing HTTP methods.

    Overview

    The HTTPMethod enumeration defines the most commonly used HTTP methods for performing network requests. Each case in this enumeration represents a specific type of HTTP request method, such as GET, POST, PUT, PATCH, and DELETE. These methods are used to interact with web servers and perform various operations on resources.

    This enumeration provides a convenient way to specify the HTTP method for a network request. By using the raw values associated with each case, developers can easily set the HTTP method of a URLRequest object.

    Usage

    To use the HTTPMethod enumeration, simply assign one of its cases to the httpMethod property of a URLRequest object. For example:

    var request = URLRequest(url: URL(string: "https://example.com")!)
request.httpMethod = HTTPMethod.get.rawValue

  • HTTPStatusCode

    An enumeration representing HTTP status codes.

    Overview

    The HTTPStatusCode enumeration defines a comprehensive list of HTTP status codes categorized into success, redirection, client error, and server error responses. Each case in this enumeration represents a specific HTTP status code and its associated integer value. The enumeration also provides utility methods to check the category of a given status code and to create an instance from an integer value.

    This enumeration is useful for handling and interpreting HTTP responses in network operations, allowing developers to easily identify the type of response and take appropriate action based on the status code.

    Usage

    To use the HTTPStatusCode enumeration, either match against specific cases or use the utility methods to check the category of a status code. For example:

    if HTTPStatusCode.isSuccessful(response.statusCode) {
    // Handle successful response
} else if HTTPStatusCode.isClientError(response.statusCode) {
    // Handle client error response
}

  • NetworkErrorCases

    An enumeration representing various types of network-related errors, each with a custom description.

    Overview

    The NetworkErrorCases enumeration defines a wide range of network-related errors, each with a specific description. These error cases cover common network issues such as invalid URLs, no data responses, timeouts, client and server errors, parsing and decoding errors, and more. Each case can provide a detailed description that can include additional context or messages to aid in understanding the nature of the error.

    This enumeration is designed to be used with the NetworkError struct to provide a standardized way to handle network errors across different parts of an application.

    Usage

    To use the NetworkErrorCases enumeration, create an instance of the NetworkError struct with a specific case and optional message. Use the description property to get a textual representation of the error case.

    let errorCase = NetworkErrorCases.badRequest("Invalid parameters")
let error = NetworkError(errorCase: errorCase, apiErrorMessage: "Parameter 'id' is missing")
print(error.detailedDescription)  // Output: "Bad Request: Invalid parameters - Parameter 'id' is missing"
